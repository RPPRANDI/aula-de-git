Olá mundo do Git e GitHub

// Git init: Inicializa um novo repositório // Git add (Git add .): adiciona arquivos para a área de stage (pronto para ser commitado) // Git commit (Git commit -m): grava as alterações feitas na área de stage como uma nova versão // Git status: verifica o status atual do repositório, incluindo arquivos modificados // Git log (Git log --all): ver histório de commits // Git checkout: para alternar entre branches, criar novas ou mudar versões // Git branch: lista das branches do projeto // Git checkout -b "nome_da_branch": para criar uma nova branch // Git merge: unir o conteúdo de duas ou mais branches em uma única branch. Geralmente de uma branch de desenvolvimento para um branch principal // Git remote add origin: Conectar o projeto local ao repositório do GitHub // Git branch -M main: Renomear o nome da branch // Git push -u origin: Enviar o código do terminal ao GitHub // Git push: comando para enviar ao GitHub // Gi clone: Pegar o repositório do Git e baixar na máquina apontando a URL do projeto.
